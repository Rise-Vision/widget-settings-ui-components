URL Field

An Angular directive or jQuery plugin that allows a user to enter a URL.

Angular

Dependencies Angular JS Angular Bootstrap Storage Selector Tooltip

The directive requires the use of the url attribute which configures the binding of the value for the URL.

Self-validation occurs on the input field value and if deemed invalid an error message displays. Additionally, a checkbox is provided to either opt in or out of validation.

When using the component within a form, add the valid attribute and an arbitrary model value (eg. ng-model="urlentry" in order to update your forms $validity pertaining to a valid URL entry

Additionally, the Storage Selector component is available to pick a file from Storage, which fills the input field with the URL of the file selected. In order for the Storage Selector to work properly, the URL Field directive provides the company-id and storage-type attributes so their values can be bound to the Storage Selector directive. The ID for the currently selected Company can be found in the URL where the ID is everything to the right of "company=", for example:

http://rva.risevision.com/#COMPANY_MANAGE/company=XXXXXXXX-XXXX-XXXX-XXXXXXXXXXXX

There are a number of optional attributes for further customizing the use of URL Field. You can optionally hide the label for URL Field by using the hideLabel attribute like so hide-label="hideLabel". Similarly, the Storage Selector can be hidden using the hideStorage attribute like so hide-storage="hide". If you need to listen for a blur event associated with the input field of URL Field, the component fires an event named urlFieldBlur. You can specify two different file types that the URL should validate against - image or video - for ensuring a valid file extension is entered in the URL. Use the fileType attribute like so file-type="image" or file-type="video" to do so. Lastly, in the scenario that you set a default value of "" for the URL Field but you do not want the component to initially show the invalid controls, add the init-empty attribute.

Dependencies N/A

The jQuery plugin version provides public methods getUrl(), setUrl(), and validateUrl() to use as you need.

Upon calling validateUrl(), if deemed invalid a checkbox is provided to either opt in or out of validation.

Note: An error message is not displayed in the jQuery version.
